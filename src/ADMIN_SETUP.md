# Инструкция по доступу к админке PARTA

## Доступ к админ-панели

Для доступа к админ-панели необходимо войти в систему с одним из следующих email:
- **admin@parta.app**
- **tpycbi@gmail.com**

При входе с одним из этих email автоматически активируется админский доступ, и в навигации появится кнопка "Админка".

## Возможности админки

### 1. Модерация жалоб

Админка отображает все жалобы пользователей в трёх категориях:
- **Новые жалобы** - требуют рассмотрения
- **Рассмотренные** - жалобы, которые были просмотрены но не решены
- **Решённые** - жалобы, по которым были предприняты действия

Для каждой жалобы отображается:
- Email пользователя, подавшего жалобу
- Имя пользователя, на которого подана жалоба
- Причина жалобы
- Дополнительные детали (если указаны)
- Дата и время подачи жалобы

### 2. Действия с жалобами

Для новых жалоб доступны действия:
- **Рассмотрено** - отметить жалобу как рассмотренную (без блокировки пользователя)
- **Заблокировать** - заблокировать пользователя, на которого подана жалоба

### 3. Управление заблокированными пользователями

Во вкладке "Заблокированные" отображается список всех заблокированных пользователей с информацией:
- Имя и email пользователя
- Причина блокировки
- Дата блокировки

Для каждого заблокированного пользователя доступна кнопка **Разблокировать**.

## Технические детали

Все данные о жалобах и заблокированных пользователях хранятся в Supabase KV Store:
- Жалобы: ключ `reports`
- Заблокированные пользователи: ключ `blocked_users`

## Создание жалобы

Пользователи могут подать жалобу во время активной сессии, нажав на кнопку "Пожаловаться".

Доступные причины жалоб:
- Неподобающее поведение
- Спам или реклама
- Отвлекающее поведение
- Другое

Жалобы автоматически отправляются на сервер и становятся доступны в админке.
